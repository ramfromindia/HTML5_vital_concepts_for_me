HTML5 Best Practices — DOs & DON'Ts

Practical guidance for writing high-quality HTML5 and collaborating with CSS3 and JavaScript.

DOs

Prioritize semantics, accessibility, maintainability, and performance.

  • Semantic Structure: Use <header>, <main>, <article>, <section>, <nav>, <footer>.
  • Separation of Concerns: Keep HTML for structure, CSS for presentation, JS for behaviour.
  • Accessibility First: Add alt, <label>, proper roles, and keyboard focus support.
  • Mobile-First / Responsive: Include meta viewport, use relative units, Flexbox/Grid.
  • Progressive Enhancement: Ensure core content works without JS; enhance when available.
  • Performance Mindset: Minimize DOM depth, lazy-load images, compress assets.
  • Consistent Naming: Use clear class names or conventions (BEM, utility scales).
  • Lint & Validate: Use validators and linters (W3C, HTMLHint) and include checks in CI.
  • Document Contracts: Describe data-* attributes and custom events for collaborators.
  • Feature Detection: Prefer feature-detection over user-agent sniffing.

DON'Ts

Avoid legacy patterns, accessibility pitfalls, and brittle markup.

  • No Presentation in HTML: Avoid inline styles and layout in tables.
  • No Inline Handlers: Don't use onclick or other inline JS handlers.
  • Don't Overuse Divs: Prefer semantic elements to generic <div>/<span>.
  • Don't Rely on Color Alone: Ensure information conveyed by color also has text or icons.
  • Don't Block Rendering: Avoid blocking synchronous resources in the critical path.
  • No Deprecated Tags: Avoid <font>, <center>, and obsolete attributes.
  • No Duplicate IDs: Keep IDs unique; use classes for repeated selectors.
  • No Missing A11y: Don't ship interactive controls without keyboard/screen-reader support.
  • No Hardcoded Locale Text: Design for internationalization from the start.

Working With CSS3

  • External Stylesheets: Keep CSS in external files for caching and separation.
  • Scoped Selectors: Target classes/component roots; avoid global tag selectors that bleed styles.
  • CSS Variables: Use custom properties for theme tokens and design scales.
  • Respect Preferences: Honor prefers-reduced-motion and system color schemes.

Working With JavaScript

  • Unobtrusive Behavior: Attach listeners via addEventListener, not inline attributes.
  • Modular Code: Use ES modules or bundlers and avoid leaking globals.
  • Sync State & ARIA: Update ARIA properties when UI state changes.
  • Graceful Fallbacks: Provide server-rendered content or non-JS fallbacks for critical flows.

Quick Examples

Use these small patterns in your projects.

Correct Button

<button type="button" class="btn-primary">Save</button>

Deferred Script

<script src="/js/app.js" defer></script>

Accessible Form

<label for="email">Email</label>
<input id="email" name="email" type="email" required aria-describedby="emailHelp">
<small id="emailHelp">We’ll only use this to contact you.</small>
